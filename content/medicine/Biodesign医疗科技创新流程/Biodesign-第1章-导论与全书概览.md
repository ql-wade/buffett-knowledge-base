# 第1章 导论与全书概览

## 核心观点

### Need First：需求驱动，不是技术驱动

医疗创新的最大杀手不是技术不行，而是**在解决一个没人真正需要的问题**。Biodesign的核心立场：所有创新从 Need 开始，不是从技术开始。

> **关键区分**：
> - Technology-Push（技术推动）：我有技术 → 找应用场景
> - Need-Pull（需求拉动）：我有需求 → 找技术方案
> Biodesign 100% 站在 Need-Pull 一边。

### 三阶段框架：Identify → Invent → Implement

```
Identify（识别）  →  Invent（发明）  →  Implement（实施）
"找对问题"           "设计最优方案"      "把方案送到患者手中"
```

三个阶段不是线性瀑布，而是有反馈回路：
- Invent 发现技术不可行 → 回到 Identify 重新选需求
- Implement 发现报销无望 → 回到 Invent 换方案

### 创新是可训练的技能，不是天赋

斯坦福 Biodesign Fellowship 的数据证明：经过系统训练的团队，创新成功率远高于凭直觉做事的团队。1998年至今培养了数千人，孵化100+公司，累计创造收益超21亿美元。

## 操作方法

### Biodesign 全流程速查表

| 阶段 | 步骤 | 核心问题 | 关键产出 |
|------|------|---------|---------|
| **Identify** | Need Finding | 临床痛点在哪里？ | 100+ candidate needs |
| | Need Screening | 哪些值得投入？ | 六维矩阵筛选到1-3个 |
| | Need Scouting | 深度验证需求 | 10-20位医生访谈 |
| | Need Specification | 精确定义需求 | Need Statement |
| **Invent** | Brainstorming | 有什么解决方案？ | 100+ 想法 → 3-5个概念 |
| | Prototyping | 方案可行吗？ | 三级原型 |
| | IP Strategy | 能保护吗？ | 专利组合 |
| **Implement** | Regulatory | 如何获批？ | FDA路径选择 |
| | Reimbursement | 谁来付钱？ | Coding-Coverage-Payment |
| | Business Model | 怎么赚钱？ | 商业模式设计 |
| | Company Formation | 怎么运营？ | 团队+融资+里程碑 |
